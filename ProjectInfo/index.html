Preparations

We took a look at one or two implementations of the 'battlefield' game already implemented on the net. Seeing them we decided that a) they were too slow and b) they were too ugly. So we decided that we needed to implement the game in java.

The job

After that basic design decision we got working. Unfortionatly the stuff we got working on is completley unrelated to Naval Warefare, since we both got more-than-fulltime jobs.

The main Java program

We have split the javacode into two packages, sea and naval, with the partition defined as 'everything that migth be used in for example a minesweeper' in sea and everything that is realy specific to Naval Warefare in naval. The reason for this is that it should make the code cleaner and easier to understand, but also that I (Rasmus) plan to write that game, eventually. We also considered making a package (misc or something) for realy generic classes, but havent found the need for those classes yet. It might still arrive, however.

The applet used on the 'one player game' page is naval.OnePlayer . It is a simple panel with a Label for status messages and the actual game, naval.Naval as Components.

The game

When the user clics the mouse in the game area Sea.mouseUp gets called by awt. It calculates the Coord that corresponds to the mousclic's position and calls Naval.touch (since Naval extends sea). . . Do I really have to get this detailed? There are source code, you know...

The Future of NW

By no means are we finished just because the MUMS project deadline has arrived. During the whole process, Rasmus and I became so fanatical about NW, that we plan on continuing with the project in order to offer a cool, extremely high-bandwith game onto the Internet.
